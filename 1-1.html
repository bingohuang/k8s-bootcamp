技能学习：Kubernetes 集群介绍

‹ 后退


Kubernetes 用于协调一个高可用的计算机集群，这些计算机连接在一起看作成一个可工作的单元。 无需你指定某个特定的机器，就可以将容器化应用部署到集群当中。 为了充分使用这种新型的部署方式，应用需要从单一的主机中解耦出来，重新打包，并容器化。过去的打包方式，应用打包后就被直接安装在指定的机器上，往往和宿主机绑定的很深，而容器化应用可以做到更灵活和好用。
总的来说，Kubernetes 可以在集群中以一种更高效的方式，分发和调度容器化应用，它本身也是一个开源平台，并且已经用于生产级了。

一个 Kubernetes 集群由两种节点类型组成：

  • Master节点： 用来协调集群
  • Nodes 节点： 用来运行应用

总结:

  • Kubernetes 集群
  • Minikube

Kubernetes 是一个生产可用的开源平台，它被用于在集群上编排和运行应用容器。


集群示意图


Master节点用于管理整个集群，它会协调集群中的所有动作，比如调度应用、维护应用状态、扩容应用以及滚动更新等。

Node节点作为集群中的一个工作机器，一般是一台虚拟机或者物理机。 每一个node需要安装一个Kubelet，Kubelet相当于一个agent，用于管理节点并和master通信。 Node也需要安装容器操作的工具，比如Docker或rkt。生产级的 Kubernetes 集群最少要有3个节点。

Master节点用于管理集群，而Node节点用于运行应用。

当你在 Kubernetes 上部署应用时，就是让master启动应用容器。 Master 将容器调度在集群node上。 Node 通过 Kubernetes API（由master暴露）和master通信，终端用户也可以通过 Kubernetes API 直接和集群交互。

Kubernetes 集群既能部署在物理机，也可以部署在虚拟机上。想要从头开始部署 Kubernetes，使用 minikube 是个好办法。Minikube是一个轻量级的 Kubernetes 实现，它可以在你的本机上创建一个VM，并在这之上部署一个包含一个节点（既是Master也是Node）的集群。Minikube 支持 Linux、Mac OS 和 Windows。Minikube CLI 提供了一些基本的操作来和你的集群交互，包括启动、停止、查看和删除（命令行中直接输入minikube就可以看到帮助说明）。 在这个训练营中，会提供一个预装好minikube的在线终端环境，这样你都不需要安装任何环境，就可以直接体验 Kubernetes 集群，是不是很酷呢！
